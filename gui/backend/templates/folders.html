How to read these results
Each folder corresponds to one automated benchmark execution.

Every run is labelled by five dimensions: controller, topology, metric group, measurement mode, and scenario (e.g., baseline, malformed traffic or REST faults).

  • Controller: SDN control plane under test.
  • Topology: emulated data plane in Mininet.
  • Metric group: what is being measured (e.g., D-ITG).
  • Mode: reactive vs proactive flow installation.
  • Scenario: normal operation vs injected faults.

Inside each folder, the dashboard aggregates CSV logs and processed statistics into interactive tables and charts. These visualisations support the discussion on dependability and performance in the thesis.
